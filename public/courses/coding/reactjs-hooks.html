React.js: React Hooks (Dasar useEffect)

Memahami useEffect Hook untuk mengelola side effects dalam komponen React

Pengenalan React Hooks

extensionApa itu React Hooks?

• Fungsi khusus yang memungkinkan menggunakan state dan fitur React lainnya tanpa menulis class
• Diperkenalkan di React 16.8
• Selalu dimulai dengan kata "use" (useState, useEffect, dll)

lightbulbMengapa Hooks Diperkenalkan?

• Menyederhanakan kode dan memudahkan sharing stateful logic
• Menghindari "wrapper hell" dari Higher-Order Components
• Memungkinkan penggunaan state dalam functional components

Class vs Functional Components

check_circleClass: lifecycle methods (componentDidMount, dll)
check_circleFunctional: Hooks (useEffect untuk side effects)
check_circleFunctional: lebih ringkas dan mudah dibaca

Apa itu useEffect?

settingsDefinisi useEffect

• Hook yang memungkinkan menjalankan side effects dalam functional components
• Menggantikan lifecycle methods: componentDidMount, componentDidUpdate, dan componentWillUnmount
• Sintaks: useEffect(() => { ... }, [dependencies])

scheduleKapan Menggunakan useEffect?

• Saat perlu sinkronisasi dengan sistem eksternal
• Saat perlu menjalankan kode setelah render
• Saat perlu membersihkan resources sebelum komponen unmount

Cara kerja useEffect

codeSintaks Dasar useEffect

// Sintaks dasar
useEffect(() => {
  // Side effect logic
  console.log("Effect runs");
  
  // Cleanup function (opsional)
  return () => {
    console.log("Cleanup runs");
  };
}, [dependencies]); // Array dependencies

scheduleKapan useEffect Dijalankan?

1
Mount: Setup function dijalankan setelah render awal
2
Update: Cleanup dijalankan dengan nilai lama, kemudian setup dengan nilai baru
3
Unmount: Cleanup function dijalankan satu kali terakhir

Dependency Array

linkApa itu Dependency Array?

• Parameter kedua dalam useEffect
• Mengontrol kapan effect dijalankan kembali
• Membandingkan dependencies dengan nilai sebelumnya menggunakan Object.is

Cleanup Function

cleaning_servicesApa itu Cleanup Function?

• Fungsi yang dikembalikan dari useEffect
• Bertugas membersihkan efek sebelum komponen unmount atau sebelum effect dijalankan kembali
• Mencegah memory leaks dan perilaku tidak diinginkan

scheduleKapan Cleanup Function Dijalankan?

1
Sebelum component unmount - Membersihkan resources
2
Sebelum effect dijalankan kembali - Membersihkan effect lama
3
Di development mode - Dijalankan ekstra untuk testing

buildContoh Penggunaan

useEffect(() => {
  // Setup - membuat koneksi
  const controller = new AbortController();
  
  fetch(url, { signal: controller.signal })
    .then(response => response.json())
    .then(data => setData(data));
  
  // Cleanup - membatalkan request
  return () => controller.abort();
}, [url]);

import { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Buat AbortController
    const controller = new AbortController();
    const signal = controller.signal;

    // Fetch data
    fetch('https://api.example.com/data', { signal })
      .then(res => res.json())
      .then(result => {
        setData(result);
        setLoading(false);
      })
      .catch(err => {
        if (err.name !== 'AbortError') {
          console.error(err);
        }
      });

    // Cleanup function
    return () => {
      controller.abort();
    };
  }, []); // Empty dependency array

  return loading ? 'Loading...' : 'Loaded!';
}

Best Practices dan Common Pitfalls

thumb_upBest Practices

check_circleSertakan semua dependencies yang digunakan dalam effect
check_circleSelalu gunakan cleanup function untuk subscriptions
check_circleGunakan useCallback untuk fungsi dependency
check_circlePisahkan effects dengan tujuan berbeda

warningCommon Pitfalls

errorMissing dependencies - Effect tidak berjalan saat dependency berubah
errorInfinite loops - Effect mengupdate state penyebab re-run
errorUnnecessary dependencies - Effect berjalan terlalu sering
errorNon-stable references - Objek/fungsi menyebabkan re-run

summarizeKesimpulan

useEffect adalah Hook yang powerful untuk mengelola side effects dalam React. Dengan memahami dependency array dan cleanup function, Anda dapat menghindari bugs dan memory leaks.

Ingat: "Setup and cleanup should mirror each other"
